# Contributing to LinkPaw

First off, thank you for considering contributing to LinkPaw! It's people like you that make LinkPaw such a great tool.

## How to Contribute

### Fork the Repository
All contributions should be made through a fork of the main repository. 
1. Fork the project to your own GitHub account.
2. Clone your fork locally.
3. Create a new branch for your feature or bugfix.

### Submit a Pull Request
When you're ready to share your changes:
1. Push your branch to your fork on GitHub.
2. Open a Pull Request from your fork to the `main` branch of the official LinkPaw repository.
3. Ensure your PR description clearly explains the changes and references any related issues.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, if applicable.
3. If required, update the `scripts/release.sh` file to reflect any changes to the release process.
4. The project maintainers will review your PR and provide feedback.
5. The maintainers will merge your PR once it has been approved.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.

## Want to get more involved?

If you have interst in helping to maintaing this code base or even just review PR's please let us know by opening an issue or reaching out on the project repository.

## Questions?

If you have questions or need help, please open an issue on the project repository.
